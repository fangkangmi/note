Facilitator Run-Sheet — Agent Skills for Code Review

LBG · 90 minutes · 20–25 people · hybrid (demo + hands-on) · goal: build buy-in

Pre-flight (before the room fills)

  • ☐ Dry-run done: real PR through both paths — confirmed no-skill misses the APIE violation
  • ☐ Links live: PR · APIE doc · capture sheet · sandbox PR
  • ☐ Cached recordings of both AI runs (fallback)
  • ☐ @lbg-reviewer ready to comment on a real PR
  • ☐ Chart numbers + [brackets] filled in deck

If something breaks

  • Live AI run fails → play the cached recording, keep talking.
  • No clause id prints → narrate the clause from the standard (it's model-dependent).
  • Humans found a lot → lean on consistency/auditability, not just the gap.

Run of show

ClockSegmentDo / Say / Click
00:00Open5 min Title slide. Say: “How long does a proper APIE review take you?” (take 2–3). Then hands: “Who catches every violation, every time?” — few hands = the hook. Show agenda.
05:00You review it first12 min Hand out the 3 links. Start the 10-min timer. Walk the room — note who opens the standard vs reviews from memory. No AI.
17:00Debrief6 min Collect: avg time · % who found the headline violation · most-missed. Edit the chart Data → set “Found by humans”. Land: “the gap is the point.”
23:00AI — without skill10 min Bet first: hands up — will it catch the violation? Then run live (or recording). It comments on style/naming, no clause, misses the org rule. Land: “AI alone doesn’t know your standard.”
33:00★ AI — with skill15 min Run live. Slow down. Point at the clause citation. Re-ask the bet. Then: scorecard → big number (“~10 min → ~10 sec”) → Two lenses (pass@k): “skill adds APIE depth but narrows the model’s view — that’s why we run both; both outputs are valuable.” → Why it matters: auditability first.
48:00Your turn — hands-on20 min Groups of 4–5 on the sandbox PR. Trigger the review with the skill; watch it cite APIE; try to fool it. Facilitators roam. Collect “nailed it / missed it” notes (feeds the skill + the rollout pitch).
68:00Where this goes15 min LIVE demo @lbg-reviewer — comment it on a real PR, let it post the APIE review inline (8 min). Be honest it’s a PoC. Then discussion (7 min): advisory vs block-merge? · who signs off for audit? · keeping the skill in sync as the standard evolves? · which repos/standards next? Assign an owner to each.
83:00Wrap & close7 min This week: run it on your next OpenAPI PR; send what it missed/over-flagged. Name the owner collating feedback for the rollout. Thanks + Q&A.
